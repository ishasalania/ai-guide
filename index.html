Why this matters for you

Semantic search is what makes RAG systems intelligent rather than just fast. Instead of matching exact keywords, they match meaning in vector space. So a query for "quarterly earnings" can surface documents that contain "Q3 revenue", "annual profit", or "financial results", without any keyword overlap. Azure AI Search, Microsoft Fabric OneLake, and Copilot for Microsoft 365 all use this approach under the hood.

Azure AI Search

Azure AI Search offers hybrid retrieval that combines traditional keyword search (BM25) with semantic vector search. It re-ranks results by semantic relevance, so your documents surface based on what users mean rather than exact word matches.

Microsoft 365 Copilot

When you ask Microsoft 365 Copilot "show me the deck Sarah shared about Q2", it uses semantic search across your Microsoft Graph-connected data to surface the right file, even if it's saved as "strategic_review_final_v3.pptx".

03
The architecture powering enterprise AI

RAG: how it
actually works

RAG stands for Retrieval-Augmented Generation. Instead of relying solely on what a language model learned during training, RAG retrieves relevant documents from your own data at query time and uses them to ground the model's response. This significantly reduces hallucinations and keeps answers current.

Step by step · click to explore each stage
01

The user asks a question

The user asks a question in plain language, such as "What's our refund policy?" or "Summarise the Q3 report." No special formatting is required. The system handles the complexity in the steps that follow.

02

The query gets turned into a vector

An embedding model converts the question into a high-dimensional vector, a list of numbers that represents its meaning mathematically. OpenAI's text-embedding-3-large model, for example, produces a vector with 1,536 dimensions. This numerical representation is what makes semantic search possible.

03

The vector store finds the most relevant chunks

The vector database compares the query vector against pre-indexed document chunks using a similarity metric such as cosine similarity. Azure AI Search, for example, uses an HNSW (Hierarchical Navigable Small World) index to do this efficiently at scale. The top 3 to 5 most relevant chunks are retrieved in milliseconds.

04

The chunks get injected into the LLM prompt

The retrieved document chunks and the original user question are combined into a single prompt that is sent to the language model. A system prompt instructs the model to answer only based on the provided context, not its training data, and to cite the source document in every response.

05

The model returns a grounded, cited answer

The language model generates a response grounded in the retrieved context rather than its general training knowledge. A well-configured RAG system will include a citation such as "Source: Refund Policy v2.pdf, page 3" alongside every answer. Because the response is anchored to specific documents, hallucinations are significantly reduced, though not entirely eliminated, which is why evaluation is still important.

Step-by-step for builders

How to build
RAG in 2026

You don't need a computer science background to build effective RAG systems. You do need to make the right architecture decisions from the start. Here are the seven that matter most.

1

Start with your data, not your model

The quality of a RAG system is largely determined by the quality of its source data, not the model powering it. Clean, well-structured, up-to-date documents will outperform any model advantage. Use Microsoft Fabric to establish a governed, well-structured OneLake before writing any AI code.

2

Chunk intelligently, not by character count

Poor chunking strategy is one of the most common causes of low-quality RAG outputs. Rather than splitting by fixed character count, chunk by meaningful semantic units: paragraphs, sections, or question-and-answer pairs. A chunk size of around 512 tokens with 20% overlap is a reasonable starting point. Azure AI Search's integrated vectorisation feature handles this automatically.

3

Use hybrid search, not vector-only

Pure vector search misses exact keyword matches such as product codes or proper nouns. Pure keyword search misses semantic relationships. Hybrid retrieval, combining vector search, BM25 keyword ranking, and semantic re-ranking, consistently outperforms either approach alone. Azure AI Search supports this configuration out of the box.

4

Always cite your sources in outputs

Instruct the language model to cite its source document in every response. Returning the document name, section, and page reference alongside each answer allows users to verify claims, builds trust in the system, and makes it much easier to identify when a response is poorly grounded. Microsoft 365 Copilot does this natively.

5

Evaluate continuously, not just at launch

RAG quality degrades as source documents become outdated or as user query patterns shift. Establish automated evaluation that tracks retrieval precision (are the right chunks being returned?), answer faithfulness (is the response supported by the retrieved content?), and user feedback signals. Azure AI Foundry includes built-in evaluation pipelines for all three.

6

Design for sovereignty from day one

In regulated industries, data residency and access control must be designed in from day one. Use Azure Private Link to keep all API traffic within your virtual network. Replace API keys with Managed Identity for all service-to-service authentication. For the most sensitive data, Azure Confidential Computing provides hardware-level encryption even during active processing.

7

Build a feedback loop from users to the index

Every time a user marks a response as unhelpful, that is a data point. Log it, trace it back to the chunks that were retrieved, and use it to improve your chunking strategy, update stale documents, or adjust your system prompt. RAG systems improve through this kind of structured feedback loop, not just at initial deployment.

Quickstart · build a RAG app with Azure in one day
1
Step 1 · approx 2 hours
Ingest data into OneLake (Microsoft Fabric)
Connect your SharePoint sites, SQL databases, or file storage to Microsoft Fabric. Use Fabric Data Pipelines to clean and land data in Delta Lake format within OneLake. Basic ingestion can be configured without custom code using the Fabric visual pipeline editor.
2
Step 2 · approx 1 hour
Create an AI Search index (Azure AI Search)
Use the "Import and vectorise data" wizard in the Azure portal. Point it at your OneLake data source. It handles chunking, calls the embedding model (text-embedding-3-large by default), and builds a hybrid vector and keyword index automatically.
3
Step 3 · approx 2 hours
Build your chat app in Azure AI Foundry
In Azure AI Foundry, use the Chat playground to connect GPT-4o to your Azure AI Search index. Test with real user queries from your target audience. Define a system prompt that specifies response tone, citation format, and scope boundaries.
4
Step 4 · approx 1 hour
Deploy and secure
Deploy your application from Azure AI Foundry to Azure App Service using the built-in deployment flow. Enable Managed Identity for service authentication, configure Private Link if data residency requires it, and add Entra ID (formerly Azure Active Directory) for user authentication.
5
Step 5 · ongoing
Evaluate and iterate with Foundry evals
Configure automated evaluation runs in Azure AI Foundry to track groundedness, relevance, and coherence scores on a scheduled basis. Connect user feedback signals from your application interface. Use these signals to iterate on chunking, prompting, and document freshness on a regular cycle.

Solution engineering with AI in the loop
Client-facing work includes demos, architecture reviews, and RFP responses. Before each call, I use Microsoft 365 Copilot to pull relevant context from past email threads and documents. After calls, I dictate a voice note which OpenAI Whisper transcribes, and Claude converts into a structured follow-up email draft. The whole process takes around eight minutes instead of thirty.
3
1:00 – 3:00 PM · Content
AI-assisted scripting and research
For LinkedIn or TikTok content, I brainstorm out loud with Claude for around ten minutes: a brain dump of what I want to say, key ideas, possible hooks, and a rough script structure. For carousel posts, I ask for a seven-slide outline and then write the actual copy myself. CapCut's AI features handle automatic captions and rough cuts for video.
4
3:00 – 5:00 PM · Strategy
AI handles the admin, i do the thinking
Email triage runs through Superhuman's AI prioritisation. Slide updates use Microsoft Copilot in PowerPoint. For brand partnership proposals, a structured prompt template generates a first draft in under five minutes, which I then personalise with specific brand fit reasoning.
5
Friday evening · Weekly review
One AI conversation to close the week
I paste the week's notes and key interactions into Claude and ask a few reflection questions: what patterns are visible, what I should do more of, and what I have been avoiding. It does not replace personal journalling. It gives me a structured starting point for reflection that I can then respond to in my own words.
