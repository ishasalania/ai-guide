Microsoft products like Azure AI Search, Microsoft Fabric OneLake, and Copilot for Microsoft 365 all use this under the hood.

Microsoft example: Microsoft 365 Copilot

The Work IQ layer — what is it?

Work IQ is Microsoft's intelligence layer that sits between Copilot and your data. Step-by-step for builders

How to build RAG in 2026 — best practices

You don't need a PhD. You need the right architecture decisions. Here are the ones that separate RAG prototypes that break in production from ones that scale.

1

Start with your data, not your model

The quality of your RAG system is 80% data quality, 20% model choice. Clean, well-structured, up-to-date source documents matter far more than whether you use GPT-4o or Phi-4. Use Microsoft Fabric to create a governed, clean OneLake before you touch any AI code.

2

Chunk intelligently — don't just split by character count

Bad chunking is the #1 cause of bad RAG answers. Chunk by semantic units (paragraphs, sections, Q&A pairs), not by fixed character counts. Use 512-token chunks with 20% overlap so context doesn't get lost at boundaries. Azure AI Search's integrated chunking does this automatically.

3

Use hybrid search, not vector-only

Pure vector search misses exact matches (product codes, names, IDs). Pure keyword search misses semantic matches. Hybrid search (vector + BM25 + semantic re-ranking) consistently outperforms both. Azure AI Search does this out of the box. Always enable it.

4

Always include source citations in your outputs

Tell the LLM to cite its sources in every answer. Return the source document name, chunk index, and page number alongside every generated response. This lets users verify, builds trust, and makes hallucination detection trivial. Copilot does this natively — copy the pattern.

5

Evaluate continuously — not just at launch

RAG quality degrades as your source data gets stale. Set up automated evaluation: track retrieval precision (are the right chunks being found?), answer faithfulness (is the answer supported by the retrieved chunks?), and user feedback. Azure AI Foundry has built-in evaluation pipelines for exactly this.

6

Design for sovereignty and security from day one

Especially in regulated industries: never design a system where sensitive data leaves your perimeter. Use Azure Private Link to keep all traffic internal, Managed Identity instead of API keys, and Azure Confidential Computing for the most sensitive workloads. Retrofitting security is 10× harder than building it in.

7

Build a feedback loop from users to the index

Every time a user marks an answer as "not helpful", that's a signal. Log it. Trace it back to the chunks that were retrieved. Use it to improve chunking, re-index, or flag documents for human review. This is how production RAG systems get better over time, not just at launch.

Set up automated evaluation runs in Azure AI Foundry. Track groundedness, relevance, and coherence scores. Connect user feedback. Ship improvements weekly.

Ready to use — just fill in the blanks

Prompt starter kit

These are real prompts I use for work, content, and thinking. Copy the text into Claude, ChatGPT, or Microsoft Copilot. Replace the [brackets].

For explaining complex AI concepts simply
Use in: Claude / ChatGPT / Copilot
Explain [CONCEPT — e.g. "RAG", "vector embeddings", "agentic AI"] to a smart non-technical audience. Use one clear analogy, three key takeaways, and one common misconception people have. No jargon without explanation. End with one question I can ask an expert to go deeper.
Tip: Great for turning Microsoft docs or research papers into LinkedIn posts
For building a RAG system prompt
Use in: Azure AI Foundry / Copilot Studio
You are a [ROLE — e.g. "customer support assistant"] for [COMPANY]. You have access to a knowledge base of [DOCUMENT TYPE — e.g. "product manuals, FAQs, policies"]. Answer questions ONLY based on information in the knowledge base. If you cannot find the answer, say "I don't have that information in my current knowledge base" — do not guess. Always cite the source document at the end of your answer in the format: Source: [document name, page/section].
Tip: Paste this as your system prompt in Azure AI Foundry's chat playground when connecting to AI Search
For content ideation (creator mode)
Use in: Claude / ChatGPT
I create content about [TOPIC] for [PLATFORM — TikTok / LinkedIn / Instagram]. My audience is [DESCRIBE — e.g. "students and early-career professionals interested in AI and tech"]. Give me 10 specific content ideas that are educational but instantly watchable. For each idea: the hook (first 3 seconds), the core insight, and one unexpected angle that makes it shareable. Focus on ideas that work as short-form video OR carousel posts.
For technical decision making
Use in: Claude / Copilot (with search)
I need to choose between [OPTION A] and [OPTION B] for [USE CASE — e.g. "building a RAG system for a German government client"]. Key constraints: [LIST CONSTRAINTS — e.g. "data sovereignty required, budget under €50k, team has Azure expertise"]. Give me a structured comparison covering: technical fit, cost, security/compliance, scalability, and Microsoft ecosystem alignment. End with a clear recommendation and the top 3 questions I should ask the vendor before committing.
Tip: This is my go-to prompt before recommending a solution to a client
For turning a meeting into action
Use in: M365 Copilot / Claude
Here are my notes from a [TYPE — customer call / team meeting / discovery session]: [PASTE NOTES]. Extract: 1) the 3 most important things I learned, 2) decisions that were made, 3) open questions that need answers, 4) my action items with suggested deadlines, 5) one follow-up email I should send. Format as a clean summary I can share with my manager.

How I actually use this every day

My AI workflow as a Solution Engineer + Creator

I live in two worlds — enterprise AI sales engineering at Microsoft Germany, and content creation for 1M+ people. Here's how AI threads through both, without burning me out.

My time allocation with vs without AI assistance
☀
7:00 AM · Morning intelligence
AI briefing before I touch email
I start with a 10-minute Perplexity run: "what happened in enterprise AI and Azure announcements in the last 48 hours?" + "any news about [client industry]?" This gives me talking points for client calls without spending an hour on LinkedIn. Then I ask Claude to draft my top 3 priorities for the day based on my notes from yesterday.
💼
9:00 AM – 12:00 PM · Client work
Solution engineering with AI in the loop
Demos, architecture reviews, RFPs. I use M365 Copilot to pull context from past email threads before every call ("summarise my last 3 emails with this client and what we agreed"). After calls, I dictate a voice note → Whisper transcript → Claude turns it into a structured follow-up email draft. I edit and send. Total: 8 minutes instead of 30.
🎥
1:00 PM – 3:00 PM · Content creation
AI-assisted scripting and research
For long-form LinkedIn or TikTok: I brainstorm out loud with Claude for 10 min (brain dump → key ideas → hook options → script draft). For carousels: I ask for a 7-slide structure, then write the actual words myself. For video: I never read off a script — I use the AI output as a knowledge prompt, then talk naturally. CapCut AI does captions and cuts.
📊
3:00 PM – 5:00 PM · Strategy & admin
AI handles the admin, I do the thinking
Email triage with Superhuman AI. Slide updates with Copilot in PowerPoint ("update this slide with the Azure AI Foundry announcement from last week"). Brand deal proposals: I use a template prompt that generates a first draft in 3 minutes, then personalise the one thing that matters — the brand fit rationale.
🌙
Evening · Weekly review
One AI conversation to end the week
Every Friday: I paste my week's notes into Claude and ask "what patterns do you see? What should I do more of? What am I avoiding?" It doesn't replace journaling — it gives me a starting point for reflection. Then I set 3 intentions for next week and add them to Notion AI.
